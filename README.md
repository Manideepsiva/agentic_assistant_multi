# Agentic Multimodal Assistant

A deployed, agentic application that accepts **text, images, PDFs, and audio —
together in a single request** — extracts everything, understands the goal,
and autonomously plans and executes the minimal tool chain to accomplish it.
If the goal is ambiguous, it asks **one** clarifying question before acting.
All final outputs are text-only.

**Stack:** FastAPI · LangGraph + LangChain · **Groq** (Llama 3.3 70B for
planning/tools/synthesis, Llama 4 Scout for vision OCR fallback, Whisper
Large v3 Turbo for audio — all free tier, no credit card) · Tesseract (OCR
with confidence) · **React (Vite)** frontend with a live plan-trace rail.

---

## 1. What it can do

| # | Task | How |
|---|------|-----|
| 1 | Image/PDF text extraction | Tesseract OCR with **confidence %**; Llama 4 Scout vision fallback for hard images (stylised fonts, code screenshots); pypdf text layer with OCR fallback for scanned pages |
| 2 | YouTube transcript fetching | URLs are detected in **any** input (query, PDF text, OCR output, audio transcript); transcript fetched with a graceful fallback message |
| 3 | Conversational answering | Friendly grounded Q&A over the combined context |
| 4 | Summarization | Always the mandatory 3 formats: 1-line + 3 bullets + 5-sentence |
| 5 | Sentiment analysis | Label + confidence + one-line justification |
| 6 | Code explanation | Language detection, explanation, bug warnings, time complexity |
| 7 | Audio transcription + summary | Whisper (Groq) → cleanup → 3-format summary, with duration |
| 8 | Cross-input reasoning | Compares/combines multiple inputs to answer one unified query |


## 2. Project layout

```
app/
  main.py               FastAPI app: /api/chat (JSON) + /api/chat/stream (SSE), session store,
                         mounts the built React app (static/) at "/"
  config.py              Groq-only settings, .env auto-loaded
  schemas.py              Pydantic models: plans, trace events, responses
  ingestion/
    pipeline.py           Type router; graceful per-file degradation
    image_ocr.py           Tesseract (confidence) → Llama 4 Scout vision fallback
    pdf_parser.py           pypdf text layer → OCR fallback; URL discovery
    audio_transcriber.py     Groq Whisper (whisper-large-v3-turbo) + cleanup
  agent/
    graph.py              LangGraph: plan → (clarify | execute → synthesize)
    prompts.py              Planner + synthesizer prompts
    state.py                 Typed agent state
    llm.py                    ChatGroq factories: get_llm, get_planner_llm (JSON mode), get_vision_llm
    cost.py                    Pre-execution cost estimator
  tools/
    registry.py            Tool registry = single source of truth for planner + executor
    text_tools.py            summarize / sentiment / explain_code / answer / compare
    web_tools.py              YouTube transcript + generic URL fetch
frontend/                 React (Vite) source — see "Frontend" below
  src/App.jsx, components/, api.js, index.css
static/                   Built frontend output (generated by `npm run build`; served by FastAPI)
Dockerfile                Multi-stage: Node builds the frontend, Python serves everything
```

## 3. Run locally

Prereqs: Python 3.11+, Node 18+, plus system packages `tesseract-ocr` and
`poppler-utils` (only needed for image OCR and scanned-PDF fallback — or just
use Docker, which includes them).

### Backend

```bash
git clone <repo-url> && cd agentic-assistant
cp .env.example .env            # add your GROQ_API_KEY (free — console.groq.com)
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8000   # .env is loaded automatically
```
